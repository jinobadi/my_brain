### 1. 수익화 모델 비교 분석 (지노바디AI MVP 기준)

| 모델 | 장단점 | MVP 적합도 | 전략적 판단 |
|:---|:---|:---|:---|
| **구독제 (Subscription)** | 예측 가능한 현금 흐름, 지속적 가치 제시 필요. 이탈 관리 부담 큼. | ⭐⭐⭐ | 로컬 AI는 업데이트/모델 개선 지속성이 핵심. 구독제가 장기적 생존에 유리하나, MVP 초기에는 진입 장벽으로 작용할 수 있음. |
| **일회성 라이선스 (One-time)** | 전환 장벽 최소화, 초기 현금 흐름 확보 용이. 지속 수익 발생 안 됨. | ⭐⭐ | 1인 개발자로서 초기 유입과 피드백 수집이 시급함. 일회성 판매 시 가격 탄력성 테스트에 유리함. |
| **혼합형 (구독 + 평생 라이선스)** | 구독의 지속성 + 일회성의 낮은 진입 장벽 동시 확보. 관리 복잡도 ↑ | ⭐⭐⭐⭐ | **가장 권장.** MVP 검증 단계에서는 구독을 표준으로 하되, 초기 유입용 '일회성 평생 라이선스' 프로모션을 병행해 가격 민감도와 전환 동선을 측정해야 함. |

**💡 현빈의 권고:** MVP 1개월 목표(`수익형 상품 구조 공부`)에 부합하려면 **구독 티어를 기본 구조로 두고, 일회성 라이선스를 '가설 검증용 프로모션'**으로 운영하는 혼합형으로 진화하세요. 매출 규모보다 '어떤 결제 유형이 어떤 페르소나에서 더 높은 전환을 보이는지'를 데이터로 뽑는 것이 이번 달의 성공 기준입니다.

### 2. MVP 검증 핵심 KPI 정의

| KPI | 정의 | MVP 검증 기준 (Pass/Fail) | 측정 방법 |
|:---|:---|:---|:---|
| **전환률 (Landing → Install)** | 랜딩 방문자 중 무료 설치/로그인 완료 비율 | ✅ Pass: ≥15% / ⚠️ Fail: <10% | Supabase Auth 로그 + 이벤트 추적 |
| **무료 사용자 전환율 (F2P)** | 무료 사용자 중 유료 결제 완료 비율 | ✅ Pass: 3~5% / ⚠️ Fail: <1% | PayPal/Stripe 테스트 모드 결제 로그 |
| **이탈률 (Churn)** | 결제 후 30일 내 구독 취소 또는 결제 실패 비율 | ✅ Pass: <5%/월 / ⚠️ Fail: >10%/월 | 결제 게이트웨이 실패 로그 + 구독 상태 변경 |
| **활성화 지점 (Activation)** | 로컬 WASM 엔진 첫 성공 추론 완료 시간/빈도 | ✅ Pass: 설치 후 24시간 내 3회 이상 사용 | 로컬 로컬 로그(JSON/SQLite) → Supabase Sync |

**📌 검증 원칙:** MVP 단계에서는 `F2P 3% 미만` 시 가격 인하보다는 **가치 제시(CTA/후크/기능 매칭)** 재검토를 우선합니다. `Churn 10% 초과` 시 결제 주기(월/연) 또는 환불 정책 재설계가 필요합니다.

### 3. 가격 전략 초안 & 1개월 검증 로드맵

#### 🔹 가격 안 (Draft v1)
| 티어 | 가격 | 제공 범위 | 타겟 페인포인트 해결 |
|:---|:---|:---|:---|
| **Lite** | `$9/월` | 로컬 AI 기본 추론 월 100회, 커뮤니티 지원 | 예산 민감한 개인 사용자, 실험적 사용 |
| **Pro** | `$19/월` | 무제한 로컬 추론, API 연동, 우선 업데이트 | 실무 생산성 확보가 필수인 프리랜서/개발자 |
| **Lifetime** | `$99` (프로모션 한정) | 로컬 영구 라이선스, 12개월 기술 지원 | "구독 피로" 해소, 일회성 투자 선호자 |

> 💡 **전략적 근거:** `$9`는 심리적 진입 장벽을 낮추고 구독 습관을 형성합니다. `$19`는 실무자 기준 "커피 한 잔 이상"의 생산성 ROI를 가정합니다. `$99` 일회성은 초기 현금 흐름과 가격 민감도 테스트용 베이스라인입니다.

#### 🗓️ 1개월 검증 로드맵 (구조 공부 → 데이터 정제)
| 기간 | 핵심 과제 | 산출물/체크포인트 |
|:---|:---|:---|
| **W1 (05/20~26)** | 결제 플로우 테스트 모드 연동 + KPI 대시보드 구축 | PayPal/Stripe 테스트 환경 배포. Supabase에 `events`, `subscriptions`, `payments` 테이블 설계. |
| **W2 (05/27~06/02)** | 초기 50명 베타 유입 + A/B 테스트 시작 | 구독($19) vs 일회성($99) 프로모션 전환률 비교. 랜딩 훅 2안 A/B 테스트. |
| **W3 (06/03~09)** | 이탈 요인 분석 + 가격 민감도 테스트 | 환불/결제 실패 원인 추출. 20% 할인 코드 제공 시 F2P 변화 측정. |
| **W4 (06/10~16)** | 데이터 정제 & 최종 모델 판정 | `F2P ≥3%` 또는 `Churn ≤5%` 달성 시 구독 모델 지속. 미달 시 일회성 전환 또는 가격 재조정. |

**🎯 1개월 목표 부합 조건:** 실제 매출액이 아닌 **`결제 전환 동선 데이터의 품질`**과 **`가설 검증 완료 여부`**를 성공 기준으로 삼습니다. 수익형 상품의 구조를 "공부"하는 것이 목표이므로, 결제 실패 로그와 CTA 클릭 데이터의 정제가 우선됩니다.
